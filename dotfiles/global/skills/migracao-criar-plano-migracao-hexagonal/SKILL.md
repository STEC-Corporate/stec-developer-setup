# criar-plano-migracao-hexagonal

## Quando usar

Apos analise de viabilidade hexagonal; necessidade de plano detalhado para execucao.

## Documentos

.cursorrules, resultado da analise S20.

## Passos

1. Usar resultado da analise de viabilidade (portas/adaptadores identificados).
2. Definir etapas por modulo ou camada; ordem que preserve testes e contratos.
3. Para cada etapa: arquivos a alterar, testes a atualizar, checklist de validacao.
4. Garantir que Clean Architecture seja preservada apos migracao.

## Saida

Plano detalhado (etapas, arquivos, testes). Pronto para executar-migracao-hexagonal.

## Pre-condicoes
- Escopo e contexto da tarefa definidos.
- Arquivos-alvo identificados.
- Criterios de aceite minimos estabelecidos.

## Checklist de saida
- Resultado entregue e verificavel.
- Evidencias objetivas registradas.
- Riscos e proximos passos documentados.

## Cobertura de stack e dominio
- Stacks suportadas: Java, Kotlin, Python, TypeScript, React, React Native, NextJS, NestJS.
- Dominios: backend, frontend e mobile.
