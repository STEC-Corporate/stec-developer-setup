# mapear-endpoint-template-bff-api

## Quando usar

Novo fluxo ou feature vindo do Template ou BFF. Duvida sobre qual microsservico implementa um fluxo.

## Documentos

Docs/PREMISSA-TEMPLATE-BFF-API.md, Docs/CONSOLIDADO_FUNCIONALIDADES_FRONTEND.md, Docs/microsservicos.md.

## Passos

1. Identificar fluxo ou feature.
2. Consultar CONSOLIDADO para requisitos de backend.
3. Consultar microsservicos.md para servico(s) do dominio.
4. Listar endpoint(s) esperados e microsservico(s).
5. Indicar lacunas se CONSOLIDADO pedir algo nao mapeado.

## Saida

Lista de microsservicos e endpoints. Contratos esperados. Lacunas se houver.

## Pre-condicoes
- Escopo e contexto da tarefa definidos.
- Arquivos-alvo identificados.
- Criterios de aceite minimos estabelecidos.

## Checklist de saida
- Resultado entregue e verificavel.
- Evidencias objetivas registradas.
- Riscos e proximos passos documentados.

## Cobertura de stack e dominio
- Stacks suportadas: Java, Kotlin, Python, TypeScript, React, React Native, NextJS, NestJS.
- Dominios: backend, frontend e mobile.
