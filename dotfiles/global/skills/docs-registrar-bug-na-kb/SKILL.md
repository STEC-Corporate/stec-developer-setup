# registrar-bug-na-kb

## Documentos

docs/knowledge-base/ (ex. errors.md), Docs/error-codes-reference.md.

## Passos

1. Preencher Descricao do erro (mensagem, cenario, ambiente).
2. Preencher Motivo do erro (causa raiz).
3. Preencher Solucao (passos ou codigo que corrige).
4. Opcional Prevencao (teste ou checklist).
5. Entregar texto pronto para colar em docs/knowledge-base/errors.md ou equivalente.

## Saida

Entrada formatada para a base de conhecimento. Nao escrever no arquivo sem aprovacao.

## Quando usar
- Use quando a tarefa estiver alinhada ao objetivo da skill.
- Use quando for necessario padrao tecnico e repetibilidade.

## Pre-condicoes
- Escopo e contexto da tarefa definidos.
- Arquivos-alvo identificados.
- Criterios de aceite minimos estabelecidos.

## Checklist de saida
- Resultado entregue e verificavel.
- Evidencias objetivas registradas.
- Riscos e proximos passos documentados.

## Cobertura de stack e dominio
- Stacks suportadas: Java, Kotlin, Python, TypeScript, React, React Native, NextJS, NestJS.
- Dominios: backend, frontend e mobile.
